CROSS-CUTTING PATTERNS
================================================================================

PATTERN 1: FIRE-AND-FORGET MUTATIONS (12 instances)
  Mutations that await Supabase but do not check {error} response.
  Files: SupportTickets.tsx (3), SalesPipeline.tsx (3), useVendorSubmissions.ts
  (2), useServiceRequests.ts (1), useCustomVendors.ts (1), useMarkSignalRead.ts
  (1), useActiveBanner.ts (1).
  Root cause: Missing const { error } = await pattern.

PATTERN 2: PREMATURE SUCCESS TOAST (7 instances)
  toast.success() called before or without verifying Supabase response.
  Files: SalesPipeline.tsx (3), useRolePermissions.ts (3+), Configure.tsx (1).
  Root cause: Toast triggered immediately after await, not after error check.

PATTERN 3: OPTIMISTIC WITHOUT ROLLBACK (5 instances)
  Local state updated before DB, no revert on failure.
  Files: SupportTickets.tsx (3), useRolePermissions.ts (3), useCustomVendors.ts
  (1).
  Counter-example: useUserPreferences.ts implements correct rollback.

PATTERN 4: STUB HOOKS FEEDING LIVE UI (34 hooks, 13+ pages)
  Hooks return hardcoded [] or null. Consumer pages render full UI with
  zero data. No user-facing indication of incomplete feature.
  Highest-risk: SchedulePage, IncidentReportsPage, BonusDashboardPage,
  InventoryPage, CallbacksPage.

PATTERN 5: CHAINED MUTATIONS WITHOUT TRANSACTION (3 instances)
  Multiple Supabase .update()/.insert() calls without RPC transaction.
  Files: useVendorSubmissions.ts (2), useServiceRequests.ts (1).
  Root cause: PostgREST does not support multi-statement transactions.
  Fix: Wrap in Supabase RPC function for atomicity.
